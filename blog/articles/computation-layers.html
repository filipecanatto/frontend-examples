As Principais Camadas de Um Computador

Como tudo na vida um computador também pode ser dividido em partes para facilitar seu estudo e diminuir sua complexidade. Pode dividir o computador em três partes: aplicação de usuário, sistema operacional e hardware. Nesse artigo vou sintetizar cada uma dessas camadas para que o leitor tenha uma visão superficial de como o computador esta organizado.

Programas de usuário

São programas usados pelo usuário final, seja ele um economista organizando seu trabalho em um editor de planilhas, ou um engenheiro civil construindo um projeto de um novo edifício usando um DAC (desenho assistido por computador), ou um programador escrevendo seus programas em um editor de texto. Esse é motivo pelo qual os computadores existem, para prover suporte a diversas áreas através de programas de usuário. Esses programas são criados usando uma linguagem de programação, que sucintamente falando são maneiras de instruir o hardware a fazer alguma coisa.

Sistema operacional

Quando criamos nosso programa ele não funciona de forma independente, ele sempre vai interagir com o sistema operacional para pedir suporte, mas oque seria exatamente esse suporte?. São funções já prontas disponíveis em memória que podem ser usadas pelos programas de usuário. Funções que implementam funcionalidades de propósito geral como escrever dados em dispositivo de saída, ler dados de um dispositivo de entrada, alocar memória, gerenciar a memória para que cada programa tenha sua área reservada, ou seja, que nenhum programa leia dados de outro programa, ou escreva em uma posição de memória que está sendo usada por outro programa.

Chamadas de Sistema

Quando um programa solicita uma função do sistema operacional ele faz isso através de uma chamada de sistema, quando isso acontece o programa para de executar e o sistema operacional entra em cena. O programa pode interagir diretamente com o sistema operacional ou através de uma biblioteca padrão disponibilizada junto com o sistema operacional, no linux temos a glibc (GNU library C).

O Kernel

O componente principal do sistema operacional é o kernel (do inglês núcleo), é nele que ficam implementadas todas as interfaces das chamadas de sistemas, e também é ele que gerencia os drivers e a memoria. Na figura acima foi usada um figura que mostra a arquitetura de um S.O linux porem o conceito de kernel esta presente em todas as arquiteturas de sistemas operacionais.

Hardware

Todas as camadas acima da camada do hardware só podem ser visualizadas através de abstrações e diagramas, mas o harware é algo físico podendo ser visto, tocado e sentido. É ele o responsável por executar as instruções criadas pelo programador através de uma linguagem de programação. O Hardware do computador é composto de vários componentes, sendo os principais entrada/saída ,processador e a memoria. O hardware realiza algumas operações básicas para qualquer computador, são elas: entrada e saída de dados, processamento e armazenamento. A interação entre esses componentes é basicamente da seguinte forma: o processador obtém dados e instruções da memoria, os dispositivos de entrada guardam dados na memoria e os dispositivos de saída leem dados da memoria e apresentam o resultado da computação para o usuário através de um monitor ou para outro computador através de uma placa de rede (o hardware responsável por conectar um computador a uma rede computacional). Os cursos de ciência da computação e analise de sistemas não se aprofundam muito no estudo do hardware, geralmente o mais profundo que chegamos é até portas lógicas. " Dispositivos de hardware são construídos a partir de pacote integrados de portas lógicas. E essas portas, por sua vez, podem ser construídas de portas lógicas elementares como a Nand ou Nor, é claro que cada uma dessas portas consiste de alguns dispositivos de switching. tipicamente implementados por transistors. e cada transistor é construído de - Bem, nós não iremos mais longe, porque é onde a ciência da computação acaba e a física começa. " (grifo do autor). A frase acima esta no livro Building a Modern Computer from First Principles. Isso significa que detalhes do hardware não estão no escopo de estudo da ciência da computação, sendo assim não precisamos saber como certas tarefas são implementadas, podemos apenas saber oque elas fazem e assim consumir seus serviços.

Conclusão

Espero que esse artigo ajude colegas a ter uma visão mais abrangente do computador, e não velo apenas como uma caixa preta.
